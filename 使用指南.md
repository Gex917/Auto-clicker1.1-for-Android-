## 一、编译 APK

### 方法一：使用 Android Studio（推荐新手）

**第 1 步：下载 Android Studio**

访问官网下载：https://developer.android.com/studio

**第 2 步：安装 Android Studio**

- 双击下载的安装包
- 按照默认设置一路点击 "Next"
- 安装完成后启动 Android Studio

**第 3 步：打开项目**

- 启动 Android Studio
- 选择 **"Open an existing Android Studio project"**
- 浏览到 `G:\mimo-work\AutoClicker` 文件夹
- 点击 **"OK"**

**第 4 步：等待同步**

- 首次打开会自动下载依赖包
- 等待底部进度条完成（可能需要 5-10 分钟）
- 看到 "BUILD SUCCESSFUL" 表示同步成功

**第 5 步：编译 APK**

- 点击菜单栏 **Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**
- 等待编译完成
- 点击右下角的 **"locate"** 链接打开 APK 所在文件夹
- APK 文件位置：`AutoClicker\app\build\outputs\apk\debug\app-debug.apk`

---

### 方法二：使用命令行编译

**第 1 步：打开命令提示符**

- 按 `Win + R`，输入 `cmd`，回车

**第 2 步：进入项目目录**

```bash
cd G:\mimo-work\AutoClicker
```

**第 3 步：执行编译命令**

```bash
gradlew.bat assembleDebug
```

**第 4 步：等待编译完成**

- 首次编译会自动下载 Gradle 和依赖包
- 看到 "BUILD SUCCESSFUL" 表示成功
- APK 位于：`app\build\outputs\apk\debug\app-debug.apk`

## 二、安装应用

### 第 1 步：传输 APK 到手机

**方法 A：数据线传输**
1. 用 USB 数据线连接手机和电脑
2. 在手机上选择 "传输文件" 模式
3. 在电脑上打开手机存储
4. 将 `app-debug.apk` 复制到手机任意位置（如 Download 文件夹）

**方法 B：微信/QQ 传输**
1. 在电脑上登录微信或 QQ
2. 将 APK 文件发送给 "文件传输助手" 或自己
3. 在手机上接收并保存

**方法 C：网盘传输**
1. 将 APK 上传到百度网盘/阿里云盘
2. 在手机上下载

### 第 2 步：安装 APK

1. 在手机上打开文件管理器
2. 找到并点击 `app-debug.apk`
3. 如果提示 "禁止安装未知来源应用"：
   - 点击 **"设置"**
   - 找到 **"允许来自此来源的应用"** 并开启
   - 返回继续安装
4. 点击 **"安装"**
5. 等待安装完成，点击 **"打开"**

## 三、开启权限

### 权限一：无障碍服务（必须）

**作用：** 让 App 能够执行自动点击操作

**开启步骤：**

1. 打开手机 **"设置"**
2. 找到 **"无障碍"** 或 **"辅助功能"**
   - 不同手机位置可能不同：
     - 华为/荣耀：设置 → 辅助功能
     - 小米/红米：设置 → 更多设置 → 无障碍
     - OPPO/realme：设置 → 系统设置 → 无障碍
     - vivo：设置 → 快捷与辅助 → 无障碍
     - 三星：设置 → 辅助功能
3. 找到 **"已安装的服务"** 或 **"下载的服务"**
4. 找到 **"自动点击器"**
5. 点击进入，开启开关
6. 弹出确认对话框，点击 **"允许"**

---

### 权限二：悬浮窗权限（必须）

**作用：** 让 App 能够显示悬浮控制面板

**开启步骤：**

1. 打开手机 **"设置"**
2. 找到 **"应用管理"** 或 **"应用"**
3. 找到 **"自动点击器"**
4. 点击 **"权限"** 或 **"应用权限"**
5. 找到 **"悬浮窗"** 或 **"显示在其他应用上层"**
6. 开启开关

**快捷方式：**

在 App 主界面，点击 **"开启"** 按钮，会直接跳转到设置页面。

## 四、获取屏幕坐标

### 方法一：使用指针位置（推荐）

**第 1 步：开启开发者选项**

1. 打开手机 **"设置"**
2. 找到 **"关于手机"** 或 **"我的设备"**
3. 连续点击 **"版本号"** 7 次
4. 提示 "您已处于开发者模式" 即可

**第 2 步：开启指针位置**

1. 返回设置主页
2. 找到 **"开发者选项"**
   - 华为/荣耀：设置 → 系统和更新 → 开发人员选项
   - 小米/红米：设置 → 更多设置 → 开发者选项
   - OPPO/realme：设置 → 系统设置 → 开发者选项
   - vivo：设置 → 系统管理 → 开发者选项
3. 找到 **"指针位置"** 或 **"显示触摸操作"**
4. 开启开关

**第 3 步：查看坐标**

1. 打开你要自动点击的游戏
2. 手指触摸屏幕上的目标位置（如"开始"按钮）
3. 屏幕顶部会显示：
   - `X: 540` ← 这是横坐标
   - `Y: 1800` ← 这是纵坐标
4. 记录下这两个数字

**第 4 步：关闭指针位置**

- 用完后记得关闭 "指针位置"，否则屏幕上会一直显示触摸轨迹

---

### 方法二：使用 App 布局分析

1. 打开 App
2. 点击右上角菜单
3. 选择 **"布局分析"**
4. 手指触摸屏幕上的目标位置
5. 会显示该位置的坐标信息

## 五、配置点击位置

### 添加点击位置

1. 打开 **"自动点击器"** App
2. 在 **"📍 点击位置"** 区域，点击 **"➕ 添加点击位置"**
3. 输入坐标：
   - **X**：横坐标（水平位置）
   - **Y**：纵坐标（垂直位置）
   - **等待**：点击后等待多少毫秒（1000毫秒=1秒）

### 坐标说明

```
屏幕坐标系：
(0,0) ────────────→ X (右)
  │
  │
  │
  ↓
  Y (下)
```

- **X 值**：从左到右，0 到屏幕宽度（如 1080）
- **Y 值**：从上到下，0 到屏幕高度（如 2400）

### 常见分辨率参考

| 手机分辨率 | 屏幕中心 | 常见按钮位置 |
|-----------|----------|-------------|
| 1080 x 2400 | (540, 1200) | 底部按钮约 Y=2200 |
| 1440 x 3200 | (720, 1600) | 底部按钮约 Y=2900 |
| 720 x 1600 | (360, 800) | 底部按钮约 Y=1450 |

### 删除点击位置

- 点击每个位置右侧的 **🗑️ 删除按钮**

## 六、设置执行参数

### 参数说明

| 参数 | 说明 | 建议值 |
|------|------|--------|
| **重复次数** | 执行几轮点击（0=无限循环） | 10-100 |
| **点击间隔** | 每次点击之间的等待时间 | 300-1000ms |
| **轮次间隔** | 每轮执行完后的等待时间 | 1000-3000ms |

### 参数调优建议

**游戏响应慢？**
- 增大 "点击间隔" 和 "轮次间隔"

**需要快速点击？**
- 减小 "点击间隔"（最低建议 200ms）

**不确定游戏需要多久？**
- 先设置较短的重复次数（如 5 次）测试
- 观察游戏反应后调整参数

## 七、启动自动点击

### 第 1 步：检查权限状态

打开 App，检查顶部的权限状态：
- ✅ 无障碍服务：已开启
- ✅ 悬浮窗权限：已开启

如果显示 ❌，请先开启对应权限。

### 第 2 步：配置完成

- 已添加至少一个点击位置
- 已设置重复次数和间隔

### 第 3 步：点击启动

点击底部的 **"🚀 启动自动点击"** 按钮

### 第 4 步：打开游戏

启动后，切换到你的游戏界面。

## 八、悬浮窗控制

启动后，屏幕上会出现悬浮窗控制面板：

```
┌─────────────────────┐
│   🎮 自动点击器      │
│                     │
│   状态：运行中       │
│   进度：3/10        │
│                     │
│  [▶ 开始] [⏹ 停止]  │
│                     │
│      [关闭]         │
└─────────────────────┘
```

### 按钮功能

| 按钮 | 功能 |
|------|------|
| **▶ 开始** | 开始执行自动点击 |
| **⏹ 停止** | 暂停自动点击 |
| **关闭** | 退出自动点击器 |

### 移动悬浮窗

- 按住悬浮窗可以拖动到屏幕任意位置

### 查看进度

- **状态**：显示当前运行状态
- **进度**：显示已完成/总次数

## 九、常见问题

### Q1: 点击没反应？

**检查清单：**
1. ✅ 无障碍服务是否已开启？
2. ✅ 坐标是否正确？（用指针位置验证）
3. ✅ 等待时间是否足够？（游戏可能响应慢）

**解决方法：**
- 重新开启无障碍服务
- 增大点击间隔
- 检查坐标是否准确

---

### Q2: 悬浮窗不显示？

**检查清单：**
1. ✅ 悬浮窗权限是否已开启？
2. ✅ App 是否正在运行？

**解决方法：**
- 去设置中开启悬浮窗权限
- 重新启动 App

---

### Q3: 点击位置不准确？

**原因：** 不同手机分辨率不同

**解决方法：**
- 用 "指针位置" 重新获取坐标
- 在游戏中实际触摸目标位置查看坐标

---

### Q4: 如何停止脚本？

**方法一：** 点击悬浮窗的 "⏹ 停止" 按钮

**方法二：** 点击悬浮窗的 "关闭" 按钮

**方法三：** 在 App 主界面点击 "停止"

---

### Q5: 需要 Root 吗？

**不需要！** 本 App 使用无障碍服务实现，无需 Root 权限。

---

### Q6: 耗电吗？

自动点击会消耗一定电量，建议：
- 连接充电器使用
- 降低屏幕亮度
- 减少不必要的等待时间

## 十、实战示例

### 示例 1：自动刷关卡

**场景：** 游戏关卡结束后，需要点击 "再次挑战" 按钮

**假设：**
- "再次挑战" 按钮坐标：(540, 1800)
- 每局游戏耗时约 30 秒

**配置：**
```
点击位置：
  1. (540, 1800) 等待 1000ms

重复次数：50（刷50次）
点击间隔：500ms
轮次间隔：30000ms（30秒）
```

---

### 示例 2：自动点击两个按钮

**场景：** 需要先点 "开始"，再点 "确认"

**假设：**
- "开始" 按钮坐标：(540, 1800)
- "确认" 按钮坐标：(540, 1200)

**配置：**
```
点击位置：
  1. (540, 1800) 等待 1000ms
  2. (540, 1200) 等待 1000ms

重复次数：100
点击间隔：500ms
轮次间隔：2000ms
```

---

### 示例 3：无限循环挂机

**场景：** 需要一直自动点击

**配置：**
```
点击位置：
  1. (540, 1800) 等待 500ms

重复次数：0（无限循环）
点击间隔：1000ms
轮次间隔：5000ms
```

**注意：** 无限循环时，需要手动点击 "停止" 按钮结束

## 💡 使用技巧

1. **先测试再正式运行**
   - 先设置较小的重复次数（如 5 次）测试
   - 确认无误后再增加次数

2. **保留足够等待时间**
   - 游戏需要时间响应
   - 等待时间太短可能导致点击无效

3. **定期检查**
   - 即使是挂机，也建议偶尔查看
   - 防止游戏弹窗或其他意外情况

4. **记录坐标**
   - 把常用游戏的按钮坐标记下来
   - 下次使用时直接输入
